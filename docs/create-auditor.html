Concept: An Auditor is a containerized sidecar that performs "Active Interrogation". It acts as a firewall for protocols like Model Context Protocol (MCP).

1 Scaffold Project

Initialize a compliant Auditor project using the Lucid SDK. This sets up the cryptographic binding for TEE communication.

2 Implement Policy

Define the interspect_tool_call() function. This example enforces a Model Context Protocol (MCP) allow-list.

3 Map to Schema

Map your audit results to the governance_results JSON block to ensure the log is indexable.

4 Build Container

Package the Auditor into an OCI-compliant image.

5 Sign Image

Critical: Sign the image with your Root Principal key. The Lucid Platform rejects unsigned Auditors.

6 Simulate Run

Run a local simulation with a mock Subject Agent to verify the JSON output structure.
