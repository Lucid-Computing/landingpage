How it fits together

01.

Configure Hardware

Establish the Trusted Execution Environment (TEE). This hardware-sealed boundary isolates code execution from the host, creating a "Sovereign Sidecar" for your model.

02.

Create Auditor

Deploy a custom Auditor sidecar within the TEE. The auditor enforces your specific governance logic, intercepting and sanitizing all agent interactions.

03.

Verify Agents

Connect to the Verifier Service via the Verification API. The service continuously validates the TEE's integrity, issuing cryptographically signed Verified Tokens.
